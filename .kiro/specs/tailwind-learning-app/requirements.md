# TailwindCSS学習アプリ 要件定義書

## 概要

TailwindCSS学習アプリは、初心者から上級者まで段階的にTailwind CSSを学習できるインタラクティブなWebアプリケーションです。リアルタイムプレビュー機能により、ユーザーは入力したクラス名の効果を即座に確認できます。

## 用語集

- **TailwindCSS**: ユーティリティファーストのCSSフレームワーク
- **学習レベル**: 入門編、中級編、応用編、文字組編、実践編の5段階
- **レッスン**: 各レベル内の個別の学習課題（各レベル20問）
- **プレビュー**: ユーザーが入力したクラス名の効果をリアルタイムで表示する機能
- **ヒント機能**: 正解のクラス名を表示する補助機能
- **学習ガイド**: TailwindCSSの基本ルールや制約を説明するセクション

## 要件

### 要件1: 学習レベル選択機能

**ユーザーストーリー:** 学習者として、自分のスキルレベルに応じた学習コンテンツを選択したい。

#### 受け入れ基準

1. WHEN ユーザーがアプリにアクセスしたとき、THEN システムは5つの学習レベルボタンを表示する
2. WHEN ユーザーが学習レベルボタンをクリックしたとき、THEN システムは選択されたレベルの最初のレッスンを表示する
3. WHEN 学習レベルが変更されたとき、THEN システムは現在のレッスン番号を0にリセットし、入力フィールドをクリアする
4. WHEN 学習レベルが選択されているとき、THEN システムは選択されたボタンを視覚的にハイライト表示する
5. WHERE レスポンシブデザインが必要な場合、THEN システムはモバイル端末でもボタンが適切に表示される

### 要件2: インタラクティブ学習機能

**ユーザーストーリー:** 学習者として、TailwindCSSクラスを入力して即座に結果を確認したい。

#### 受け入れ基準

1. WHEN ユーザーがクラス名を入力フィールドに入力したとき、THEN システムはリアルタイムでプレビューを更新する
2. WHEN ユーザーが正しいクラス名を入力したとき、THEN システムは正解メッセージを表示する
3. WHEN ユーザーがヒントボタンをクリックしたとき、THEN システムは正解のクラス名を入力フィールドに薄く表示する
4. WHEN ユーザーが再挑戦ボタンをクリックしたとき、THEN システムは入力フィールドをクリアし、ヒント表示を非表示にする
5. WHEN プレビューが更新されるとき、THEN システムは強制的にコンポーネントを再レンダリングして確実に反映する

### 要件3: レッスンナビゲーション機能

**ユーザーストーリー:** 学習者として、レッスン間を自由に移動して学習を進めたい。

#### 受け入れ基準

1. WHEN ユーザーが次へボタンをクリックしたとき、THEN システムは次のレッスンに移動し、入力状態をリセットする
2. WHEN ユーザーが前へボタンをクリックしたとき、THEN システムは前のレッスンに移動し、入力状態をリセットする
3. WHEN 最初のレッスンにいるとき、THEN システムは前へボタンを無効化する
4. WHEN 最後のレッスンにいるとき、THEN システムは次へボタンを無効化する
5. WHEN レッスンが変更されるとき、THEN システムは現在のレッスン番号と総レッスン数を表示する

### 要件4: 学習ガイド表示機能

**ユーザーストーリー:** 学習者として、TailwindCSSの基本ルールや制約を理解したい。

#### 受け入れ基準

1. WHEN ユーザーがアプリを表示したとき、THEN システムは学習ガイドセクションを表示する
2. WHEN 学習ガイドが表示されるとき、THEN システムは基本ルール（デフォルト値500など）を説明する
3. WHEN 学習ガイドが表示されるとき、THEN システムはよく使う値の一覧を提供する
4. WHEN 学習ガイドが表示されるとき、THEN システムは学習のコツとヒントを表示する
5. WHERE notebookアイコンが必要な場合、THEN システムは📓アイコンを使用する

### 要件5: レベル別解説機能

**ユーザーストーリー:** 学習者として、各学習レベルで学ぶ内容と制約を理解したい。

#### 受け入れ基準

1. WHEN 学習レベルが選択されているとき、THEN システムは選択されたレベルの学習内容を表示する
2. WHEN 入門編が選択されているとき、THEN システムは基本的なクラス形式とデフォルト値500の重要性を説明する
3. WHEN 中級編が選択されているとき、THEN システムはホバー効果やトランジションのポイントを説明する
4. WHEN 各レベルの解説が表示されるとき、THEN システムは色分けされた情報ボックスで視覚的に整理する
5. WHEN 解説が表示されるとき、THEN システムは実践的なヒントとコツを含める

### 要件6: レスポンシブデザイン機能

**ユーザーストーリー:** 学習者として、様々なデバイスで快適に学習したい。

#### 受け入れ基準

1. WHEN ユーザーがモバイル端末でアクセスしたとき、THEN システムは適切にレイアウトを調整する
2. WHEN 画面サイズが変更されたとき、THEN システムは要素の配置を適切に調整する
3. WHEN タブレット端末でアクセスしたとき、THEN システムは中間サイズのレイアウトを提供する
4. WHEN デスクトップでアクセスしたとき、THEN システムは最大幅を制限して読みやすさを保つ
5. WHERE 学習ガイドが表示される場合、THEN システムはグリッドレイアウトでレスポンシブに配置する

### 要件7: 学習コンテンツ管理機能

**ユーザーストーリー:** システム管理者として、各レベルの学習コンテンツを適切に管理したい。

#### 受け入れ基準

1. WHEN 学習コンテンツが定義されるとき、THEN システムは各レッスンにタイトル、説明、ヒント、解答、プレビュー関数を含める
2. WHEN 入門編が定義されるとき、THEN システムは基本的なTailwindCSSクラス（色、スペース、サイズなど）を含める
3. WHEN 中級編が定義されるとき、THEN システムはFlexbox、Grid、インタラクション効果を含める
4. WHEN 応用編が定義されるとき、THEN システムはレスポンシブ、ダークモード、アニメーションを含める
5. WHEN 各レベルが定義されるとき、THEN システムは20問の学習課題を提供する

### 要件8: UI/UX改善機能

**ユーザーストーリー:** 学習者として、見やすく使いやすいインターフェースで学習したい。

#### 受け入れ基準

1. WHEN 入力フィールドにプレースホルダーが表示されるとき、THEN システムは薄い青色（text-blue-400）で表示する
2. WHEN プレースホルダーが表示されるとき、THEN システムは灰色ではなく視認性の良い色を使用する
3. WHEN ユーザーがフィールドにフォーカスしたとき、THEN システムは適切なコントラストを保つ
4. WHEN ヒント機能が使用されるとき、THEN システムはプレースホルダーの代わりにヒントテキストを表示する
5. WHERE アクセシビリティが重要な場合、THEN システムは十分なコントラスト比を確保する

### 要件9: パフォーマンス最適化機能

**ユーザーストーリー:** 学習者として、快適な学習体験を得たい。

#### 受け入れ基準

1. WHEN 動的クラスが使用されるとき、THEN システムはTailwind CSSのsafelistで事前に生成する
2. WHEN プレビューが更新されるとき、THEN システムは即座に反映されるように最適化する
3. WHEN アプリが読み込まれるとき、THEN システムは必要なCSSクラスを事前に読み込む
4. WHEN ユーザーが入力を変更するとき、THEN システムは遅延なくプレビューを更新する
5. WHERE パフォーマンスが重要な場合、THEN システムは不要な再レンダリングを避ける
